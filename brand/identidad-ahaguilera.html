⚡

Offline-first real

True offline-first

La app funciona sin internet. No hay "modo offline" porque no hay "modo online". Los datos viven en el dispositivo del usuario.

The app works without internet. There is no "offline mode" because there is no "online mode". Data lives on the user's device.

🔒

Privacidad por diseño

Privacy by design

Los datos del usuario nunca salen de su dispositivo. Cifrado AES-256 obligatorio. Ni el desarrollador tiene acceso.

User data never leaves their device. AES-256 encryption mandatory. Not even the developer has access.

📦

Sin servidores

No servers

No backend, no API, no cloud. Zero DevOps. El deploy es copiar archivos. Una app, dos formatos: .exe + .apk.

No backend, no API, no cloud. Zero DevOps. Deploy is copy files. One app, two formats: .exe + .apk.

Mientras la industria corre hacia más complejidad (microservicios, containers, edge functions, AI ops), AHAguilera corre hacia menos. El stack entero cabe en un index.html.

While the industry runs toward more complexity (microservices, containers, edge functions, AI ops), AHAguilera runs toward less. The entire stack fits in an index.html.

9 Anti-patrones

AI Tells — patrones que delatan UI genérica

34 patrones detectables con grep. Marcan diseño no distintivo hecho por IA.

34 patterns detectable via grep. They mark non-distinctive AI-generated design.

1
border-l-4 border-primary → Fondo tintado o icono leading
2
gradient text (from-purple to-blue) → Color sólido
3
Glassmorphism decorativo → Solo si cumple propósito funcional
4
Big number + small label (métrica plantilla) → Contenido real, testimonios
5
Card grids idénticos (icon+heading+text repetido) → Variar layouts, mezclar tipos
6
Modal como primera opción → Agotar alternativas inline primero
7
Cards anidadas → Extraer contenido a su sección
8
text-gray-500 sobre bg-primary → Shade más oscuro del color de fondo
9
#000 o #fff puro → Tintar hacia hue brand (chroma 0.005-0.01)
10
Bounce/elastic easing → ease-out-quart / quint / expo
11
Animación de layout properties → Solo transform + opacity
12
Purple-to-blue gradients → Elegir UN color sólido como acento
13
Inter como única tipográfica → system-ui o fuente con personalidad
14
Rounded-square icon tile arriba de headings → Quitar icon tile o icono inline
15
Em dashes en copy (—) → Coma, dos puntos o paréntesis
16
Tiny uppercase tracked labels sobre cada sección → Un kicker fuerte basta
17
Monospace como shorthand "técnico" → Solo si la marca es genuinamente técnica
18
Dark mode = light mode invertido → Jerarquía propia (surface lighter = depth)
19
Centered stack hero con icon-card-grid → Asimetría, left-aligned
20
All-caps body copy → Reservar caps para labels cortos
21
Placeholder text con bajo contraste → 4.5:1 mínimo también en placeholder
22
Heavy color en estados inactivos → Inactivo = muted, activo = acento
23
Nested modales → Steps progresivos en línea
24
Sin skeleton states en carga → Skeleton > spinner-in-the-middle
25
Sin empty states que enseñen → Empty state que guía al primer paso
26
Display fonts en UI labels o botones → Sans para UI, display solo para branding
27
Sin state coverage (hover/focus/active/disabled) → Cada interactivo tiene sus 5 estados
28
Acentos saturados >80% → Desaturar a chroma < 0.08
29
Nombres genéricos ("Acme Corp", "SmartFlow") → Nombres reales o contextuales
30
Números falsos (99.99%, $100.00) → Datos orgánicos (47.2%)
31
Lorem Ipsum / placeholder text → Copy real
32
Emojis en UI → Bootstrap Icons o SVG
33
h-screen → min-h-[100dvh]
34
Icons sin label contextual → sr-only o aria-label

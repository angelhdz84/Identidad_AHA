Manifiesto

Apps que funcionan sin permiso, sin red, sin excusas.

AHAguilera desarrolla software offline-first: herramientas que respetan al usuario, sus datos y su tiempo. La identidad visual refleja esa filosofía — sistemas discretos, alta legibilidad, contraste funcional.

Principio 01

Offline-first

La app funciona sin conexión. La red es una mejora, no una dependencia. Sincronización opcional, conflicto resuelto en cliente, datos locales siempre.

Principio 02

Sin tracking

Cero analíticas invasivas. Cero SDKs de terceros que exfiltran datos. Telemetría opcional, anónima y local-first si existe.

Principio 03

Estándares abiertos

PWA, IndexedDB, Web Workers, Service Workers. Sin lock-in propietario. Sin permisos innecesarios. Sin binarios cerrados.

Construcción del monograma

Proporciones

  • Canvas
    Cuadrado 512×512 para favicon / PWA.
  • Box
    Caja interna al 70% del canvas, centrada. Padding lateral 15%.
  • Radius
    Esquinas redondeadas 20% del lado de la caja (rx=72 sobre 360).
  • Letras
    Tres letras A-H-A, font-weight 800, font-size 42% del lado de caja.
  • Spacing
    Letter-spacing negativo -5% del font-size. Sin kerning extra.
  • Baseline
    Letras a 70% de la altura de la caja (desde arriba).

Clear space & tamaño mínimo

El clear space mínimo es la altura de la letra "A" del monograma, en los cuatro lados. No situar elementos tipográficos o visuales dentro de esa zona.

Tamaño mínimo digital: 24px de caja. Tamaño mínimo print: 8mm. Por debajo, usar la versión "A" sola en lime como avatar circular.

No estirar, no rotar, no aplicar sombras, no aplicar opacidad < 30%, no poner sobre imágenes sin overlay. No recortar el monograma bajo ninguna circunstancia.

El modo claro usa una superficie beige cálida (#F5F0EB) con el mismo acento lime.

Paleta completa

Reglas de contraste

  • Texto
    Body text siempre --text-primary sobre --surface. Ratio AA mínimo 4.5:1.
  • OK
    Lime sobre dark = 13.2:1. Apto para texto grande y CTAs.
  • OK
    Teal sobre lime = 6.8:1. Apto para badges, no para body.
  • No
    Lime sobre beige. Contraste insuficiente (2.1:1). Usar teal.
  • No
    Verde sobre dark para texto. Sólo como acento iconográfico.

Semántica de color

  • Teal
    Institucional, identidad. Header de secciones formales.
  • Lime
    Acción principal. Botones primarios, links activos, focus rings.
  • Verde
    Confirmación, success, sync OK, validación de form.
  • Rojo
    Error destructivo. Borrar, eliminar, fallo crítico.
  • Grey
    Meta, descripciones, placeholders. Nunca para acciones.
Tipografía

Inter para todo. JetBrains Mono para código.

La marca usa una única familia sans-serif — Inter — en toda la UI, con pesos 400/500/600/700/800. Para código, tokens y metadata se usa JetBrains Mono. Esta elección garantiza consistencia visual entre el logo (donde se simula system-ui bold) y el body text, con fallback a system-ui en cualquier dispositivo.

Display — Inter 800
AHAguilera offline-first
El peso 800 con letter-spacing negativo (-2px) es la voz principal de la marca. Se usa en hero, títulos H1/H2 y el wordmark.
font-family: 'Inter', system-ui, sans-serif · font-weight: 800 · letter-spacing: -0.04em
UI Body — Inter 400/500
El peso 500 para UI labels y énfasis.
El peso 400 es la voz por defecto del cuerpo de texto. Es legible a 14-16px, soporta feature settings de variantes, y mantiene su personalidad en cualquier dispositivo vía fallback a system-ui. Se recomienda activar font-feature-settings 'cv02','cv03','cv04','cv11' para una apariencia más geométrica.
font-family: 'Inter', system-ui, sans-serif · font-weight: 400/500 · line-height: 1.6
Mono — JetBrains Mono
const offline = true
JetBrains Mono se reserva para código, snippets, tokens CSS, nombres de archivo y metadatos técnicos. No se usa en body text, títulos o UI labels. Pesos 400 (regular) y 600 (bold).
font-family: 'JetBrains Mono', 'Fira Code', 'Courier New', monospace

Sistema de diseño

Componentes discretos. Contraste funcional.

Voz de marca

Directo, técnico, sin poses.

La voz de AHAguilera habla al desarrollador que估值 el tiempo. Frases cortas. Vocabulario técnico preciso. Sin marketing inflamado. Sin promesas vacías. La marca promete offline-first y lo demuestra con cada interacción.

Hacer

Error message
"No se pudo sincronizar. Los cambios quedan guardados localmente."
CTA button
"Crear proyecto" — verbo de acción concreta.
Empty state
"Aún no hay proyectos. Crea el primero — funciona sin conexión."
Tooltip
"Los datos se guardan en este dispositivo. Opcional: sincronizar cifrado."
Release note
"v2.1 — Service Worker reescrito. Sync 3× más rápido en LTE."

Evitar

Error message
"Oops! Algo salió mal 😞. Nuestro equipo ha sido notificado."
CTA button
"¡Comienza tu viaje ahora!" — marketing inflado.
Empty state
"Tu aventura comienza aquí ✨"
Tooltip
"Tus valiosos datos están seguros con nosotros 🔒"
Release note
"¡Actualización increíble! Hemos mejorado tu experiencia."

Reglas de tono

  • Sí
    Segunda persona singular ("tú"). Trato directo, no formal.
  • Sí
    Vocabulario técnico cuando aplica: sync, cache, IndexedDB, service worker.
  • Sí
    Verbos de acción concreta: crear, guardar, sincronizar, exportar.
  • No
    Emojis en UI de producto (sí permitidos en marketing social).
  • No
    Adjetivos subjetivos: increíble, fantástico, revolucionario.
  • No
    Frases pasivas. "Se guardó el archivo" → "Archivo guardado".
  • Tech
    Para comunicaciones técnicas (release notes, docs),允许 inglés sin traducir.
Sugerencias visuales

Ideas para extender la marca.

Propuestas para llevar la identidad más allá del logo: patrones, motion, micro-interacciones, mockups. Cada sugerencia es opcional y compatible con el sistema base.

01

Patrón "grid teal + lime"

Rejilla de 12×12 celdas teal con el monograma AHA disperso en lime al 8% de opacidad. Para fondos de hero, secciones de transición y wrappers de cards. Tile SVG de 80×80.

background: url('pattern-grid.svg'); opacity: 0.08;
02

Motion "service worker pulse"

Cuando un SW sincroniza, un anillo lime pulsa 1.2s alrededor del monograma en el header. Usa @keyframes pulse-ring con box-shadow. Refuerza "offline-first" visualmente.

@keyframes pulse-ring { 0% { box-shadow: 0 0 0 0 rgba(207,244,52,0.4) } 100% { box-shadow: 0 0 0 16px transparent } }
03

Cursor "sync dot"

Pequeño círculo verde de 6px que sigue al cursor durante operaciones de sync o save. Se desvanece al completar. Detalle minimalista que comunica estado sin banners.

.sync-cursor { width:6px; height:6px; border-radius:50%; background:#07D16E; transition: opacity 0.3s }
04

Estilo "terminal mono"

Para docs técnicas y changelogs: fondo surface puro, texto JetBrains Mono 13px, con prompt lime $ y argumentos en blanco. Refuerza la voz técnica.

$ aha build --offline --target=pwa
05

Empty states ilustrados

Para tablas y listas vacías: composición geométrica simple — tres cuadrados teal redondeados en perspectiva, con un AHA lime en el central. Estilo minimalista, sin personajes.

SVG inline, 240×120, colores de marca
06

Color tokens semánticos

Extender la paleta con roles funcionales: --sync-active (lime), --sync-error (red), --offline-badge (teal). Centraliza estados en design tokens.

--sync-active: var(--accent); --offline: var(--surface-teal);
07

Favicon dinámico por estado

Intercambiar favicon.svg en runtime según estado de la app: idle (teal), sync activo (lime parpadeante), error (red). Comunicación silenciosa del estado en el tab del navegador.

document.querySelector('link[rel=icon]').href = '/favicons/favicon-sync.svg'
08

Esqueleto de carga "shimmer teal"

Skeleton screens con gradiente animado de teal → teal-hover → teal en lugar del clásico gris. Refuerza identidad mientras carga. Duración 1.8s, easing ease-in-out.

background: linear-gradient(90deg, #024653 0%, #035f6e 50%, #024653 100%); background-size: 200% 100%
09

Marca de agua institucional

Para PDFs, contratos y docs formales: monograma en blanco al 6% de opacidad como marca de agua diagonal. Encima, footer con "AHAguilera — Desarrollo Freelance — ahaguilera.dev".

opacity: 0.06; transform: rotate(-30deg); fill: #EAF4F5
